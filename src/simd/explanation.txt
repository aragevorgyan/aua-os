"loop versioned for vectorization because of possible aliasing"

void saxpy(float a, float *x, float *y) {
    for (int i = 0; i < N; i++) {
        y[i] = a * x[i] + y[i];
    }
}

The compiler thinks:

"What if x and y point to overlapping memory?"
Example:

saxpy(3, arr, arr + 1);

Version 1 (fast vectorized)

Used if pointers do NOT overlap.

Version 2 (scalar fallback)

Used if pointers might overlap.

At runtime it inserts a check like:

if (x and y do not overlap)
    run vector version
else
    run scalar version

That is called loop versioning.
